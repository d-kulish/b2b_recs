Multi-tenancy model: Each customer receives a fully isolated GCP project. No data is shared between tenants. The only shared resource is the TFX Compiler container image stored in the partner's Artifact Registry and pulled at pipeline build time via IAM.

Security: All credentials are stored in Google Secret Manager (not the Django database). Outbound connections to customer source databases use a static IP via Cloud NAT for firewall whitelisting. HTTPS is enforced on all Cloud Run endpoints.

Regions: Customer infrastructure is deployed in a single GCP region (e.g., europe-central2). Vertex AI training jobs that require GPU run in a supported region (e.g., europe-west4) with cross-region BigQuery access.
