Security & Isolation: Each tenant runs in a fully isolated GCP project. Customer data never leaves their project. Cloud NAT provides a static outbound IP for firewall whitelisting at on-premise databases. All DB credentials live in Secret Manager. HTTPS is enforced on all Cloud Run services.

Vertex AI usage: Pipelines (TFX orchestration), Custom Training (TensorFlow Recommenders — classical two-tower collaborative filtering), and Model Registry. No Generative AI, LLMs, or third-party AI APIs are used. Model serving runs on Cloud Run (TF Serving / ScaNN), not Vertex AI Prediction.

Customer data sources: The systems shown on the left are existing customer-managed data stores. Recs Studio does not deploy any agents or software into the customer environment. The ETL Runner (a Cloud Run Job within the isolated GCP project) connects via read-only APIs. The customer only needs to provide network access (e.g., firewall whitelisting of the Cloud NAT static IP). ETL setup can be performed by the customer or with our support.
