Data flow: Customer sources → ETL Runner (with Cloud NAT static IP for DB whitelisting) → BigQuery → Vertex AI TFX pipeline → TF SavedModel → Cloud Run serving endpoint. Users interact via the Django App (Cloud Run).

Multi-tenancy: Every tenant runs in their own GCP project. No shared data planes. The partner project only hosts the TFX compiler image (Artifact Registry) and TFDV parser microservice.
